How does it start: Regression Analysis Slack bot

Every Monday, Wednesday and Friday, the regression analysis slack bot will post the regression analysis pair on the #caching-rd slack channel

This pair is responsible to handle analysis of all the regression tests that are executed in that week.

The HydraDB Dashboard

Now that you are selected as the regression analysis pair, you need to access the dashboard.

This can be found at http://hydradb.gemstone.com/hdb/runs. This resource can only be accessed only if you are connected by Pivotal VPN or in a Pivotal on-premise internal network.

Ensure that the regression run that you are analyzing, has the branch as develop and user as Full-CI as can be seen in the screenshot below.

Start

Once you click on the test run it will open up a page with a report showing

  • All tests that were executed.
  • All failed and passed tests
  • Unreported failures

The Automatic Diagnoser

As shown above we can see that there are some failures that are already marked with ticket numbers in the Bugs column.

With patter matching, these failure are diagnosed automatically. The regression analysis pair does not have to analyze these failures.

Unreported Bugs

As shown above, in the Bugs column, we can see that some of the failures are marked Unreported.

Analyzing these unreported failures is the primary task of the regression pair

  • The first attempt to diagnose the failure is by using the history of the test failures. This is explained in the below linked page.
  • If the history is not help then we need to download the artifacts and analyze the failure

If you are not able to find any matching JIRAs with the failure you are trying to analyze. That means its time to create a new ticket.
